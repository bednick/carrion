# Создание мобов и зон

> **Канонический формат — [`docs/content.mobs.format.md`](../../../docs/content.mobs.format.md) и
> [`docs/content.zones.format.md`](../../../docs/content.zones.format.md).** Все ключи json — `snake_case`
> (`mob_id`, `item_id`, а не `mobID`/`itemId`). Не дублируй схему — сверяйся с этими файлами и реальными конфигами.

Моб и босс описываются **одним** форматом: босс — это просто моб, поставленный финальным боем зоны. Мультиатака,
призыв и фазы — поля самого моба, поэтому ими можно наделить любого противника.

## Где что лежит

| Сущность      | Файлы                                                            | Регистрация                                  |
|---------------|-----------------------------------------------------------------|----------------------------------------------|
| Моб / босс    | `src/mobs/<mob_id>/config.json`                                 | `src/mobs/registry.ts` (import + `MOB_CONFIGS`) |
| Зона          | `src/zones/<zone-id>/config.json`                              | `src/zones/registry.ts` (`ZONE_CONFIGS`, и см. ниже) |
| Предмет (лут) | `src/items/<item_id>/{config.json,behavior.ts,icon.svg}`        | `src/items/registry.ts`                      |
| Спрайт моба   | `public/sprites/mobs/<mob_id>/base.png`                         | автозагрузка (см. ниже), кода не требует      |

`<mob_id>` и `<item_id>` — `snake_case`; `<zone-id>` — `kebab-case` (как существующие папки).

## Новый моб

1. Создай `src/mobs/<mob_id>/config.json` по формату из `content.mobs.format.md`: `id`, `name`, `health`, `attacks[]`
   (минимум один `{ damage, interval }`). `id` обязан совпадать с именем папки. **Лут в мобе не задаётся** — он
   привязан к зоне (`mob_loot` / `boss.loot`).
   - **Пока идёт наполнение контентом, баланс не настраиваем.** Новый моб задаётся «плоскими» дефолтами, чтобы он
     был сразу читаем в бою и видно, что его ещё не балансили: рядовой моб — `"health": 10`, **босс — `"health": 20`**
     (включая его фазы-формы, чтобы бой с боссом был «толще» по всей длине), каждый удар в `attacks[]` и призыв —
     `"damage": 5`. `interval` подбирай по характеру моба. Те же значения уже стоят у всех существующих мобов.
     Реальный баланс — отдельным этапом позже.
2. Зарегистрируй в `src/mobs/registry.ts`: добавь `import xCfg from './<mob_id>/config.json'` и строку
   `<mob_id>: xCfg as unknown as MobConfig` в `MOB_CONFIGS`.
4. Спрайт: положи `public/sprites/mobs/<mob_id>/base.png`. `PreloadScene` грузит `mob-<id>` для **всех**
   `ALL_MOB_IDS` автоматически — отдельный код не нужен. Без файла моб рисуется цветным прямоугольником-заглушкой
   (бой работает), поэтому спрайт можно добавить позже (пайплайн арта — скилл `art-asset`).

## Спец-механики (любому мобу, не только боссу)

Добавляются полями в конфиг моба (примеры — в `content.mobs.format.md`):

- **Мультиатака** — несколько элементов в `attacks[]`. Каждый = независимая полоска атаки.
- **Призыв** — `summons: [{ "mob_id": "...", "count": 2 }]`. В начале боя доспавнит мобов по ссылке (берёт их
  статы; любое поле можно переопределить). Ссылается на **существующего** моба.
- **Фазы** — `phases: [{ "id": "...", "transition": "tag" }]`. Формы после «смерти», по порядку; каждая ссылается
  на моба по `id`. `transition` — тег анимации входа. Разворачивается **один уровень** (фазы у моба-формы
  игнорируются — рекурсии нет). Спрайт при переходе берётся по `mob-<id>` новой формы.

## Новая зона

1. Создай `src/zones/<zone-id>/config.json` по формату из `content.zones.format.md`: `id`, `name`, `faction`,
   `star`, `fights {min,max}`, `mob_pool: [{ mob_id, weight }]`, `mob_loot` (общий лут рядовых мобов зоны),
   `boss: { mob_id, loot }`. Босс — ссылка на моба + награда за финальный бой (фазы/призыв босса живут в конфиге
   его моба, **не** здесь). Лут (`mob_loot`/`boss.loot` → `items[].item_id`) должен ссылаться на существующий предмет.
2. Зарегистрируй в `src/zones/registry.ts`:
   - import + запись в `ZONE_CONFIGS`;
   - `id` зоны должен быть в `ALL_ZONE_IDS` (10 фиксированных слотов карты);
   - убери его из `WIP_ZONE_IDS`, иначе на карте он показан как «В разработке» и недоступен;
   - если есть фоны — добавь счётчики слоёв в `ZONE_BG_VARIANTS` (фоны — скилл `art-asset`).
3. Доступность: стартовые зоны открыты через `MetaStore` (`unlockedAreas`), остальные — покупкой пропуска
   за золото или наградой за квест.

## Привязка квестов (если у зоны есть квест-цепочка)

- Убийство любого моба/босса пишется в статистику (`MetaStore.recordMobKilled(<mob_id>)`, см.
  `ExpeditionScene.onEnemyDied`). Боссовый квест засчитывается через stat-условие: в `src/quests/definitions.ts`
  у квеста укажи `condition: { stat: 'mobs_killed', id: '<mob_id босса>' }` — `QuestSystem` сам сверит (в т.ч.
  задним числом, если игрок убил босса до выдачи квеста). Отдельную привязку в `QuestSystem` добавлять не нужно.
- Событие `EventBus.emit('boss_killed', <mob_id>)` остаётся, но только для презентации (звук победы), не для квестов.
- Сами квесты — в `src/quests/definitions.ts`. Формат и цепочки — `docs/quests.md`.

## Проверка

1. `npx tsc --noEmit` — типы и корректность json-импортов.
2. `npm run dev` и **перечислить, что проверить** — пользователь тестирует вручную в браузере
   (`http://localhost:5173`): бой с мобом, лут, босс и его механика, засчёт квеста.
3. Менял персист-схему (поля `ItemInstance` и т.п.)? Бампни суффикс ключа `carrion.meta.vN` в `MetaStore.ts`,
   чтобы сбросить несовместимые сохранения.
