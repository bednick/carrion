# Контент: оружие (слот `hand_right`)

Источник правды по слоту `hand_right`: домен, ось, формы урона (архетипы), характеристики предмета и список
предметов слота. Общие правила системы предметов (R1–R9) и функциональность слотов —
[`content.items.md`](content.items.md); боевая идентичность фракций — [`factions.md`](factions.md);
событийная модель — [`combat-events.md`](combat-events.md).

## Домен слота

**Основной урон героя.** `hand_right` — **единственный слот прямо внутри треугольника фракций**
(в отличие от `legs`, который вне боя). Каждый предмет объявляет поток атаки (`attackInterval`) и авторит урон
на `attack_ready` (см. [`combat-events.md`](combat-events.md) §5).

Оружие — **чемпион двух фракций** и намеренно не третьей:

- **Звери** (много целей / напор) — ответ: **cleave / прошив** (один замах косит волну) и **мультихит**
  (много ударов гасят свору, попутно кормят вампиризм).
- **Мародёры** (одна броняцель) — ответ: **тяжёлый пробой** (редкие сильные удары, мало роллов блока) и
  **крит** (rider `armorPierce` явно пробивает часть брони цели на крит-ударе).
- **Нежить** (бой про время) — **не контрится оружием (канон A).** Анти-нежить — это **реновация**
  (вампиризм/реген), домен другого слота. Оружие лишь **кормит** её: `fast`/мультихит = больше событий
  `damage` = больше проков лайфстила (эмерджентно, [`combat-events.md`](combat-events.md) §6). Дать оружию
  собственный анти-нежить урон = схлопнуть треугольник, поэтому запрещено.

## Ось слота

Домен — урон; ось — **профиль подачи урона**: **концентрация** (одна цель, крупный удар) ↔ **распределение**
(много целей / много ударов). Полюса оси — «весь урон в одну точку» против «размазать по доске».

Поверх оси живут **дескрипторы-риддеры**, не образующие своих осей (R1 — одна ось на слот):

- **темп** — `slow` / `fast` (интервал атаки);
- **крит** — шанс крупного удара;
- **контроль** — стан (стаки на цели → порог);
- **накопление** — раскачка урона (ramp, стаки на цели).

Архетип = **идентичность** (форма урона: набор тегов для синергий и тултипов) + **один профильный стат** (R5),
который скейлит редкость. Разные архетипы одного слота ведут себя по-разному.

### Отклонённые направления

- **Пронзание брони как отдельный узел** — избыточно: роль закрывает **крит** через свой rider `armorPierce`
  (см. «Клевец — крит» ниже) — тот и броню пробивает, и юзабелен вне Мародёров (просто менее оптимален).
- **Собственный анти-нежить урон** (оверкил-перенос / DoT сквозь фазы) — ломает треугольник. Канон A: нежить —
  через реновацию, не оружие.
- **Уклонение как инструмент оружия** — уклонение отдано **врагу-Нежити** (защита, бестелесность),
  см. [`factions.md`](factions.md). Оружию не принадлежит.
- **Анти-нежить как отдельная форма урона** — см. выше, повтор для явности: оружие покрывает 2 фракции из 3.

## Формы урона (архетипы)

Открытый веер (пополняется контентом). Столбец «движок» — что нужно для реализации; статус зависит от него.

| Архетип              | Полюс оси       | Профильный стат (R5)                    | Фракция-цель       | Движок                                     |
|----------------------|-----------------|-----------------------------------------|--------------------|--------------------------------------------|
| **Тяжёлое (burst)**  | концентрация    | урон (интервал фикс.)                   | Мародёры           | есть (`standardWeapon`)                    |
| **Сбалансированное** | центр           | урон+темп без перекоса                  | нейтр. (стартовое) | есть (`standardWeapon`)                    |
| **Лёгкое (fast)**    | к распределению | темп (интервал падает)                  | кормит вампиризм   | есть (`standardWeapon`)                    |
| **Мультихит**        | распределение   | число ударов за взмах                   | Звери / кормёжка   | есть (автор спавнит N `attack`)            |
| **Крит**             | концентрация    | шанс крита (множитель фикс.)            | Мародёры           | лёгкое (RNG в авторе, свой `behavior.ts`)  |
| **Прошив**           | распределение   | урон (охват фикс.: цель + 1 позади)     | Звери              | есть (`short_spear`, см. ниже)             |
| **Cleave**           | распределение   | урон основной цели (сплеш — фикс. доля) | Звери              | есть (`broadaxe`, см. ниже)                |
| **Раскачка (ramp)**  | концентрация    | величина стака урона                    | Мародёры           | ⏳ roadmap: система стаков (на цели)        |
| **Стан**             | контроль        | шанс / скорость накопления              | Звери / Мародёры   | ⏳ roadmap: статусы с длительностью + стаки |

> **Доска сейчас линейна** (1 ряд × 4), но враги внутри неё позиционны — у каждого есть `slot`
> (`CombatState.enemies[i].slot`), прокинутый и в `CombatView.enemies[i].slot` для предметов. Оба сплеш-архетипа
> реализованы на линейном ряду без ожидания доски 2×2: **прошив** (`short_spear`) бьёт цель + живого
> врага в строго соседней ячейке позади неё (пустая ячейка между ними блокирует эффект); **cleave** (`broadaxe`) бьёт основную цель на 100% и **всех остальных
> живых** врагов на доске сплешем (фикс. доля от урона, не полный урон). Приём одинаковый в обоих случаях —
> автор перебирает `view.enemies` и спавнит по одному `attack` на каждую цель (тот же метод, что и мультихит).
> Блокер «multi-target attack» из более раннего roadmap оказался слабее, чем считалось. Настоящая доска 2×2
> всё ещё нужна только для генуинно пространственных эффектов (удар по соседям слева/справа/по диагонали,
> разные зоны действия по рядам/колоннам) — см. [`roadmap.md`](roadmap.md).

## Характеристики предмета

Combat-статы живут в `behavior.ts` (R9), скейл — по редкости, `stats()` — единый источник правды для тултипа и
скупщика (R4). Оружие объявляет:

- **урон** (`damage`) — базовый удар; у большинства архетипов скейлит редкость.
- **интервал** (`attackInterval`, сек) — темп потока атаки; у `fast`/мультихита падает с редкостью.
- **профильный rider** (по архетипу, ровно один — R5):
    - крит: `crit_chance` (скейл) + `crit_mult` (фикс.);
    - мультихит: `hits` (число `attack` за взмах);
    - прошив: охват фиксирован (цель + 1 живой враг в строго соседней ячейке позади неё по `slot`,
      пустая ячейка между ними блокирует эффект); профильный стат — урон, как у тяжёлого;
    - cleave: `splash_ratio` — фиксированная доля урона основной цели, летящая всем остальным живым врагам
      на доске (не скейлится редкостью, скейлится только базовый урон основной цели);
    - ramp: `ramp_step` + `ramp_cap` (прирост за удар по одной цели, сброс при смене цели);
    - стан: `stun_chance` **или** `stun_buildup` + порог, `stun_duration`.

**Лестница апгрейда** — как в семействе `legs`: `common → uncommon → rare → epic` (4 ступени), скейл капается на
`epic`; `legendary` — авторская редкость (значения вручную, скейлом недостижима). Скейл — `scaleByRarity(base, r, s)`
= `base·s^index` (`src/items/scaleByRarity.ts`).

### Короткое копьё — прошив

Стартовое оружие (`short_spear`): бьёт основную цель и живого врага в строго соседней ячейке позади неё
(по `slot`, не по индексу массива — тот расходится с позицией после призывов); пустая ячейка между целями
блокирует прошив. Обе цели получают одинаковый урон. Фиксированный охват — 2 цели, не растёт со скоростью
пачки (в отличие от будущей Секиры, которая метит **всех** в ряду). Профиль — урон (скейл ×1.5), интервал фикс.

Полюс сменился с концентрации (было — «слабое тяжёлое», тизер Мародёров) на распределение: как первое оружие
игрока, теперь тизерит противостояние Зверям, а не Мародёрам — `battle_staff` остался единственным
представителем тяжёлого/концентрации, дублирования архетипов внутри слота больше нет. Директивно подтверждено
Monte-Carlo на `trampled-meadows` (рой крыс с summon) на **старых** числах (`урон 2/инт. 3.0`): при одинаковых
цифрах прошив против одиночной цели давал заметно лучший результат (в среднем на 1.1 боя дальше, HP перед
смертью выше в ~2 раза) — сама механика работает, не просто косметика.

Числа пересчитаны (2026-07) под DPS-анкор `battle_staff` (см. врезку в «Предметы» выше): `урон 5 / инт. 1.25` →
соло-DPS 4.0 (80% анкора — цена за ситуативность), против 2 целей суммарно 8.0 (160% анкора). Пересчёт
формульный, повторной Monte-Carlo проверки не делали — при необходимости точности прогнать заново.

### Боевой посох — тяжёлый пробой

Основное тяжёлое (`battle_staff`): редкий сильный удар. Профиль — урон (скейл ×1.5), интервал фикс. Апекс
одноцелевого продавливания брони; мало роллов блока по определению (низкий темп). **DPS-анкор слота**: `урон
5 / инт. 1.0` = 5.0 DPS — единственное число, реально провалидированное Monte-Carlo (`fresh-start.json`,
[`roadmap.balance.md`](roadmap.balance.md)), остальные предметы этой правки пересчитаны относительно него.

### Короткий меч — сбалансированное

Базовая линия (`short_sword`): средние урон и темп, без перекоса. Точка отсчёта и универсальный старт до сборки
профильного билда. У архетипа нет ситуативной платы — значит, нет причины быть слабее анкора: `урон 4 / инт.
0.75` → 5.33 DPS (~107% анкора; было `урон 3` → 80% анкора — недокручено без причины).

### Кинжал — лёгкое

Быстрое лёгкое (`dagger`, dual-slot `hand_right`/`hand_left`): низкий урон, короткий интервал (падает с
редкостью). Двигатель под вампиризм (частые `damage` = частые проки лайфстила) — косвенный вклад в бой с нежитью.
Раньше интервал совпадал с `short_sword` — «лёгкое» физически не было быстрее «сбалансированного», это был
баг идентичности, не только баланса. Интервал урезан (реально быстрее short_sword и battle_staff): `урон 2 /
инт. 0.5` → 4.0 DPS (80% анкора) — но 2 удара/сек против 1.33 у short_sword и 1.0 у battle_staff, и это его
настоящая валюта (частота триггеров), не голый DPS.

### Рапира — мультихит

Реализована (`src/items/rapier/`). Один клинок, быстрая серия уколов — не «два клинка в одной руке» (прежнее
название `twin_blades`/«Парные клинки» путало: оружие живёт в одном слоте `hand_right`, а название намекало на
парность). Автор спавнит **2** `attack` за один `attack_ready`, оба — в ту же цель (два честных `attack` →
два `damage` → лайфстил прокает дважды). Профиль — число ударов и темп. Чемпион против Зверей (быстро режет
свору) и сильнейший кормилец вампиризма. Пишется руками поверх идеи `standardWeapon`, но с ручным `on.attack_ready`
(два `spawn` вместо одного).

Оба удара по той же цели (не по двум разным, как прошив) — платит частотой триггеров, не лишней целью, поэтому
целится в тот же диапазон, что и «лёгкое»: `урон 2 ×2 удара / инт. 0.8` → 5.0 DPS (~анкор, без ситуативного
дисконта — тут нет бонуса «вторая цель бесплатно», только гарантированный дубль лайфстила).

### Клевец — крит

Реализован (`src/items/war_pick/`). Тяжёлое с крит-rider: с шансом наносит крупный удар (`crit_mult` фикс.
×2.0), профиль (R5) — `crit_chance` (скейл ×1.35 по редкости; урон и интервал **фиксированы**, не скейлятся —
ровно один профильный стат на архетип). **Пробивает броню цели** явным rider'ом `armorPierce: 0.5` на крит-ударе
(`GameEvent.attack/damage.armorPierce`, читает `CombatEngine.enemyDefend`) — на этот удар эффективная броня цели
режется вдвое, поэтому анти-Мародёр; но **не обходит блок/уклонение** (полное гашение по шансу) — частичная
контра. Юзабелен и против прочих, просто менее оптимален. Риск дисперсии гасит связка `heavy + crit` (редкий
крупный удар и так в теме тяжёлого). Движок: RNG в авторе (`ctx.rng()`), свой `behavior.ts`.

> **История механики пробития (2026-07).** До перехода брони мобов на %-модель (`docs/mechanics.md` §«Броня vs
> щит») пробитие было побочным эффектом флэт-вычитания: крупный удар минус фиксированное число оставлял
> пропорционально больше, чем маленький удар минус то же число — отдельного rider'а не требовалось. После
> перехода на процентное снижение (одинаково режет любой урон вне зависимости от размера) это преимущество
> исчезло бы без явного `armorPierce` — добавлен, чтобы сохранить документированную анти-Мародёр identity
> клевца (R6), не как новая ось, а как rider поверх существующего крита.

Числа: `урон 4 (фикс.) / инт. 1.1 (фикс.), крит 20%/×2.0` → средний DPS `4·(0.8 + 2·0.2) / 1.1` ≈ 4.36 (87%
анкора) — чуть ниже battle_staff уже на common, а дальше отстаёт всё сильнее (у него скейлится только шанс
крита, не урон) — цена за счёт-в-среднем ниже: пробитие брони и сама механика крита ценны сами по себе.

### Секира — cleave

Основная цель получает полный урон, **все остальные живые враги на доске** — фикс. долю (`SPLASH_RATIO = 0.3`,
т.е. 30% урона основной цели), не только сосед по линии. Профиль (R5) — урон основной цели (скейл ×1.4);
сплеш-доля фиксирована и не растёт с редкостью — так предмет не превращается в мультихит-переросток на
поздних тирах. Чемпион против Зверей, R6-ворота по своре: чем гуще пачка, тем больше суммарный урон за взмах
(в отличие от прошива, который всегда бьёт ровно 2 цели). Дом — `armor-dump` (зона Мародёров, не Зверей — по
R10 [`content.items.md`](content.items.md): анти-Зверский чемпион добывается в чужой зоне, а не в той, где он
и так чемпион).

Реализовано тем же приёмом, что прошив (перебор `view.enemies`, автор спавнит по `attack` на каждую живую
цель) — на **старых** числах (`урон 3/инт. 2.5`) Monte-Carlo на `trampled-meadows` подтвердил: сплеш даёт
заметный, но более скромный прирост, чем 100%-прошив (HP перед боссом выше, но не кратно) — ожидаемо: доля
меньше, зато охват не ограничен двумя целями.

Числа пересчитаны (2026-07) под DPS-анкор: `урон 5 / инт. 1.5` → соло-DPS 3.33 (67% анкора — самый большой
дисконт в слоте, самое медленное и committal оружие), каждая доп. живая цель добавляет 1.0 DPS (30% от 5).
На полной доске (4 врага) суммарный DPS = 3.33 + 3×1.0 = 6.33 (127% анкора) — сильнейшее оружие слота против
толпы, слабейшее один-на-один. Пересчёт формульный, повторной Monte-Carlo проверки не делали.

### Булава — стан

Стан-шкала копится с ударов (`stun_buildup` → порог) **или** флэт-шанс за удар (`stun_chance`); на срабатывании
враг не действует `stun_duration` секунд (не капает свой поток атаки). Душит напор Зверей и ответку Мародёра
(стан = нет окна блока/контрудара); нежить не контрит (воскрешение цело — канон A). Профиль — шанс/скорость
накопления. **Ловушка баланса:** `fast` + стан-по-шансу = лок насмерть; рычаги — короткий стан, высокий порог,
затухающая отдача. **Босс — стан-резист** (иначе фазы ломаются). **Требует статусов с длительностью + стаков** —
см. [`roadmap.md`](roadmap.md).

## Судьба старых предметов (решено)

Ни один hand_right-предмет на удаление не идёт (в отличие от `legs`, `head`) — все четыре
остаются по механике и месту в архетипах, но переименованы и перерисованы: старые id/названия намекали на
испорченность предмета (обломок, ржавый, гнутый, безымянное древко), хотя редкость уже сама скейлит силу
предмета внутри одного и того же id — «испорченная» версия как отдельная сущность не нужна.

- `shaft` → `short_spear` («Древко» → «Короткое копьё») — не только рескин: архетип сменился с тяжёлого
  (концентрация) на прошив (распределение), см. подробности выше.
- `broken_staff` → `battle_staff` («Обломок посоха» → «Боевой посох») — основное тяжёлое.
- `bent_blade` → `short_sword` («Гнутый клинок» → «Короткий меч») — сбалансированное.
- `rusty_knife` → `dagger` («Ржавый нож» → «Кинжал», через промежуточное «Короткий нож»/`short_knife`) —
  лёгкое, dual-slot сохраняем.

## Порядок реализации

Все пункты, для которых движок был готов или лёгок, реализованы: **Прошив** (`short_spear`), **Cleave**
(`broadaxe`), **Крит** (`war_pick`), **Мультихит** (`rapier`). Единственный оставшийся архетип — **Стан**
(`mace`) — ждёт статусов с длительностью (+ стаки), см. [`roadmap.md`](roadmap.md). `Ramp` (`flail`/«Цеп»)
исключён из активной проработки этого слота — перенесён в [`roadmap.md`](roadmap.md) как задел на будущее
вместе с системой стаков, от которой зависит.

## Открытые вопросы

1. **Число потоков атаки.** Одно оружие = один поток стамины. Мультихит — 2 `attack` на один взмах (не второй
   поток). Второе оружие (`hand_left`) — отдельный поток. Свести в UI полосок стамины при сборке билда.
2. **Дома предметов — решено по R10 (крест-накрест, не по своей фракции).** `content.items.md` §R10:
   тяжёлое/крит (анти-Мародёры) → зона Нежити `dead-fields` (`battle_staff`, `war_pick`); прошив/cleave/
   мультихит (анти-Звери) → зона Мародёров `armor-dump` (`short_spear`, `broadaxe`, `rapier`). Нейтральные
   (`short_sword`, `dagger`, брони/щиты) остались в домашних зонах своей фракции. `mace` дома пока не
   получил (не реализован) — и его цель по матрице двойная (Звери/Мародёры, control-дескриптор, не чистый
   чемпион одной фракции), так что дом по R10 ещё не очевиден, решать при реализации.
3. **Точность DPS-пересчёта common-чисел (2026-07).** Пересчитаны формулой относительно анкора `battle_staff`
   (см. врезку в «Предметы»), **не** прогнаны через `npm run balance` — при необходимости точности (особенно
   `short_spear`/`broadaxe`, у которых ситуативный профиль сложнее чистого DPS) стоит валидировать симулятором.
4. **Расхождение крита на высоких редкостях.** У `war_pick` скейлится только `crit_chance` (капается 100%),
   урон и интервал фиксированы — на `epic` он объективно отстаёт от `battle_staff` (у того скейлится урон
   без потолка). Осознанная асимметрия архетипа или недосмотр — уточнить, когда дойдём до баланса вне common.
