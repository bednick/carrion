# Контент: обувь (слот `legs`)

Источник правды по слоту `legs`: политика формирования семейства обуви и список предметов слота.
Общие правила системы предметов (R1–R9) и функциональность слотов —
[`content.items.md`](content.items.md); событийная модель — [`combat-events.md`](combat-events.md).

## Домен слота

**«Длина забега ↔ добыча»** — не-боевая ось, осознанно вне треугольника фракций
([`factions.md`](factions.md)). Ботинки не влияют на бой; они задают профиль вылазки: сколько идёшь и что
уносишь.

**Домен добычи целиком у ног.** Прежний домен головы (`magic_find`, качество/количество дропа, ширина драфта)
переехал в `legs` без остатка. Тег `magic_find` — тег обуви, не головы.

### Отклонённые направления

- **Охват доски / cleave** — характеристика оружия, не ног.
- **Скорость атаки / темп** — балансная характеристика оружия.
- **Уклонение** — пересекает митигацию брони/щита.
- **«−бои без цены»** (старый скороход) — чистый баф, запрещён.

## Два полюса оси

- **Мародёры (`fight_delta > 0`)** — жадность. Цена: больше рядовых боёв = дольше забег = больше
  накопительного стока (HP переносится между боями без лечения) = выше риск гибели до босса. Выхлоп: **одно**
  измерение добычи на предмет; разные ботинки качают разные измерения.
- **Гонец (`fight_delta < 0`)** — скорость. Выхлоп: короче забег, быстрее к боссу, ниже риск. Цена — **потеря
  агентности**: награда за босса выбирается **автоматически** (случайная карта из стандартной раскладки 5),
  экран выбора не показывается. Кейс: быстро закрыть зону ради пропуска/квеста. Цена не размывается редкостью
  и не обнуляется богатым билдом — право выбора нельзя докупить.

## Правила формирования

1. **Один выхлоп на предмет.** Ботинок качает ровно одно измерение добычи (R5: один профильный стат).
2. **Скейл редкости:**
    - мародёры — `fight_delta` **фиксирован на предмете** (апгрейд не удлиняет забеги по реальному времени);
      редкость скейлит только выхлоп;
    - гонец — редкость скейлит **глубину `−боёв`** (доля длины: common −15% → epic −45%); цена (автовыбор)
      фиксирована.
    - **Лестница апгрейда = common → uncommon → rare → epic (4 ступени).** Весь скейл редкости в семействе
      капается на `epic`. `legendary` — авторская редкость спец-предметов, скейлом недостижима; если leg-предмет
      будет `legendary`, его значения прописываются вручную, а не выводятся из ступеней.
3. **Клампы:** минимум 1 рядовой бой до босса.
4. **Сайдгрейды, не лестница.** Внутри семейства нет ультимативно сильного: разные валюты (редкость / золото /
   эссенция / ширина выбора) и противоположный полюс скорости.
5. **Ощущаемость в забеге** (требование к UI): проки ботинка видны в ленте лута — вспышка апгрейда редкости у
   «мародёра редкости», тик «+X золота» у золотого и т.п. Слот не должен быть невидимкой.

### Числа выхлопов по редкости (мародёры)

`fight_delta` — доля от базовой длины зоны, **фиксирована**, редкостью не скейлится. Реализация: при старте
забега `fights = base + round(base * fight_delta)`.

Единая мастер-кривая процентных выхлопов — `scaleByRarity(0.2, rarity, 1.5)` (тот же движок, что у
`scavenger_lens`): паритет магнитуд = честные сайдгрейды, разная валюта.

| Предмет            | мета-мод         | common | uncommon | rare  | epic  | форма                                |
|--------------------|------------------|--------|----------|-------|-------|--------------------------------------|
| `marauder_boots`   | `magicFind`      | 0.20   | 0.30     | 0.45  | 0.68  | вероятность апгрейда редкости/ролл   |
| `usurer_boots`     | `gold_bonus`     | +20%   | +30%     | +45%  | +68%  | множитель на золото                  |
| `scavenger_shoes`  | `essence_bonus`  | +20%   | +30%     | +45%  | +68%  | множитель на эссенцию                |
| `bloodhound_boots` | `drop_pity_step` | +5пп   | +8пп     | +11пп | +15пп | аддитив к шансу дропа/бездропный бой |
| `appraiser_boots`  | `reward_reroll`  | 1      | 2        | 3     | 4     | число перебросов (целое)             |

Ищейка вне мастер-кривой: pity линейный (аддитив процентных пунктов), не геометрический множитель.

**Кап редкости апгрейда — epic.** `upgradeRarity` (`src/combat/loot.ts`) сейчас капает на `legendary` —
поправить кап на `epic`, иначе `magic_find` протаскивает дроп в legendary мимо правила «legendary скейлом
недостижима».

**Оговорка по золоту:** `mob_loot.gold` во всех зонах = 0 — `gold_bonus` пока работает только на карту золота
награды босса (лут-ролл золота пустой). Принято; можно насыпать mob-золото в зоны позже.

### Сапоги мародёра — «мародёр редкости»

Пересоздаётся с нуля (старый код удалён). Выхлоп: шанс апгрейда редкости найденного
(`magicFind` 0.20 / 0.30 / 0.45 / 0.68). `magic_find` должен применяться и в ленте лута
(`rollLootTable`), и в драфте награды босса (`buildRewardOptions`). Кап апгрейда — `epic` (см.
таблицу чисел).

### Сапоги ростовщика — «мародёр золота»

Выхлоп: бонус к золоту из всех источников забега (лут-ролл и карта золота в драфте), множитель
+20% / +30% / +45% / +68%. Новый мета-мод `gold_bonus`.

### Башмаки старьёвщика — «мародёр эссенции»

Выхлоп: бонус к эссенции (bump ролла эссенции в награде), множитель +20% / +30% / +45% / +68%. Новый
мета-мод `essence_bonus`. Флейвор-родня `scavenger_lens`.

### Сапоги ищейки — «мародёр удачи» (pity-дроп)

Выхлоп: каждый рядовой бой **без** дропа повышает шанс дропа на следующем. Дроп выпал = бонус в ноль, копится
заново. **Потолка нет** (мягкая→жёсткая pity: длинный сухой стрик рано или поздно даёт дроп). На награду босса
**не влияет** — только рядовой лут-ролл (`rollLootTable`).

Природа выхлопа — *стабильность*, не матожидание: режет засухи. Отдельный психопрофиль от «мародёра редкости»
(качество) и золота/эссенции (валюта). Редкость скейлит **шаг** прибавки: **+5пп / +8пп / +11пп / +15пп** за
бездропный бой (аддитив, вне мастер-кривой). Новый мета-мод `drop_pity_step`.

UI: HUD-тик «нюх +X%» растёт по боям, вспышка + сброс в ноль при дропе. Слот не невидимка.

**Зависимость реализации:** нужен существующий per-fight «упал ли предмет вообще» в `rollLootTable`, к которому
цепляется бонус. Если рядовой дроп сейчас гарантирован каждый бой — понятия «бездропный бой» нет, вешать не на
что. Проверить перед кодом.

### Сапоги оценщика — «мародёр точности» (реролл драфта)

Выхлоп: пул перебросов на экране награды босса. Реролл **1 карты за раз**, тянет из **луп-пула зоны** (того же,
что и драфт — иначе эксплойт жирной таблицы). Можно перекидывать уже реролльнутую карту. Пул тратится свободно,
хоть весь в одну карту. Заменяет прежнего «скупца» — активное действие игрока сильнее по агентности и
ощущаемости, чем пассивная лишняя карта, и не упирается в ширину экрана.

Прямой антипод гонца: гонец = автовыбор (агентность −−), обычный драфт = 0, оценщик = агентность ++. Явная
шкала выбора закрывает прежнюю претензию о неравных полюсах.

Скейл редкости — число перебросов N: **common 1 / uncommon 2 / rare 3 / epic 4**. Новый мета-мод
`reward_reroll` + кнопка реролла на карте в `buildRewardOptions`, счётчик оставшихся.

### Обмотки гонца — «гонец»

Единственный предмет минусового полюса. Редкость углубляет `−бои` — доля длины **−15% / −25% / −35% / −45%**
(в отличие от фикс-мародёров, здесь `fight_delta` скейлится редкостью). Кламп ≥1 рядовой бой до босса.
Награда босса всегда автовыбором (мета-мод `auto_reward`, экран драфта пропускается), цена фиксирована —
редкостью не размывается.

## Судьба старых leg-предметов (решено)

Все старые предметы слота `legs` удалены из кода (`deserter_boots`, `leather_greaves`, `marauder_boots`) —
слот пуст, наполняется заново по правилам этого документа. `marauder_boots` при новой реализации
«мародёра редкости» заводится как новый предмет с тем же id, а не восстанавливается из старого кода.

## Порядок реализации

1. **Сапоги мародёра** — почти бесплатно (`magic_find` живой, расширить на драфт); заодно капнуть
   `upgradeRarity` на `epic`.
2. Ростовщик / старьёвщик — новые мета-моды `gold_bonus` / `essence_bonus`.
3. **Ищейка** — мета-мод `drop_pity_step`; сперва проверить наличие per-fight шанса дропа в `rollLootTable`.
4. Гонец — `fight_delta < 0` + автовыбор награды.
5. **Оценщик** — последним (реролл-пул `reward_reroll` + кнопка в `buildRewardOptions`).

Отдельно: `fight_delta` сейчас в коде — **абсолютное целое** (`MetaModifiers.fightDelta`). Наши значения —
доля длины; при старте забега конвертировать `fights = base + round(base * fight_delta)`.

## Открытые вопросы

1. Ищейка: базовый per-fight шанс дропа, от которого стартует накопление pity (`entry.chance` в лут-таблицах
   зон сейчас задаёт дроп по-предметно — определить базу).
2. Дома предметов: в лут-таблицы каких зон класть новые ботинки (обувь внефракционна — можно размазать по
   всем трём стартовым зонам).
