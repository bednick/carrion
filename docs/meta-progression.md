# Мета-прогрессия

Прогрессия строится без уровней и очков прокачки.
Игрок продвигается через открытие областей, накопление ресурсов, расширение рецептов и улучшение снаряжения.

---

## Ресурсы

### Золото

- **Источники**: продажа предметов скупщику, награды за квесты, редкий дроп с мобов
- **Применение**: покупка пропусков в новые локации, услуги скупщика, **плата кузнецу за крафт**

### Эссенция

Материал для повышения редкости снаряжения. Тиров три: `uncommon`, `rare`, `epic`. Обычного (`common`) тира
нет — обычная эссенция упразднена; легендарного тоже нет — легендарное снаряжение не крафтится (см. «Снаряжение»).

- **Источник**: разборка предметов. Разбор даёт **4 эссенции своего тира + 1 эссенцию тира выше**:
  - `common` → **1 uncommon** (обычной эссенции нет, минимальный возврат);
  - `uncommon` → 4 uncommon + 1 rare;
  - `rare` → 4 rare + 1 epic;
  - `epic` и `legendary` → **5 epic** (тира выше epic нет — «+1» упирается в потолок).
- **Применение**: ингредиент при повышении редкости (улучшении). Не валюта — за саму операцию платят золотом.
  Повышение до тира T стоит **10 эссенций тира T**.

Оба ресурса **глобальные** — общие для всех персонажей.

---

## Прогрессия

### Карта областей

- Открытие новых зон: через квесты (убить босса → пропуск в следующую) или покупку пропуска у скупщика
- **Центральная область** открывается автоматически, когда все девять зон попали в `completed_areas`
- Пройденные области помечаются отдельно (можно перепроходить за лутом)

### Снаряжение

- Сундук накапливает всё снаряжение (лут уходит в него автоматически, рюкзака нет)
- **Три стойки для брони** — сохранённые пресеты билдов; активная стойка (`active_stand`) запоминается и
  используется при входе на локацию, её можно сменить на стенде или прямо в бою
- Основной вектор прогрессии: редкость снаряжения растёт через улучшение (1 предмет + эссенция → следующий тир)
- Стоимость улучшения = **10 эссенций целевого тира** + плата кузнецу золотом: `uncommon` — 100, `rare` — 1000,
  `epic` — 10000
- Улучшением доступны тиры до **эпического**. **Легендарное** снаряжение не улучшается — его можно только найти
  (награда за квесты / боссов высоких зон)
- **Цена продажи** скупщику = **половина** платы кузнецу за редкость предмета (`itemSellPrice` в
  `items/craft.ts`): `common` — 5, `uncommon` — 50, `rare` — 500, `epic` — 5000, `legendary` — 50000 (плата
  кузнецу для `common`/`legendary` условная — 10 и 100000, продолжает геометрию ×10 за тир, хотя сами эти
  редкости улучшением не достигаются)

## Персистентность

Хранилище: `localStorage`, ключ `carrion.meta.v1`.

> **Версионирование схемы пока не поддерживается.** Проект играется только локально, поэтому миграции старых
> сохранений мы не пишем. Несовместимые изменения формата меты (переименование/удаление полей, смена структуры)
> **допустимы без бампа версии ключа** — единственное требование: в описании правки явно перечислить, что меняется,
> и попросить вручную сбросить прогресс (`MetaStore.resetAll()` или очистка `localStorage`-ключа). Версионирование
> ключа вернём, когда появятся внешние игроки.

Все структурные ключи меты — `snake_case`, как и в json-конфигах (`completed_areas`, `unlocked_areas`,
`armor_stands`, `pending_reward`).

```json
{
  "gold": 0,
  "essence": { "uncommon": 0, "rare": 0, "epic": 0 },
  "completed_areas": [],
  "unlocked_areas": [
    "dead-fields"
  ],
  "chest": [],
  "armor_stands": [
    { "head": null, "body": null, "legs": null, "hand_left": null, "hand_right": null, "ring": null, "amulet": null },
    { "head": null, "body": null, "legs": null, "hand_left": null, "hand_right": null, "ring": null, "amulet": null },
    { "head": null, "body": null, "legs": null, "hand_left": null, "hand_right": null, "ring": null, "amulet": null }
  ],
  "active_stand": 0,
  "stats": {
    "mobs_encountered": [],
    "mobs_killed": {},
    "items_crafted": {},
    "items_carried_out": {},
    "items_discovered": [],
    "zones_entered": {},
    "zones_returned": {}
  },
  "quests": {
    "active": [],
    "pending_reward": [],
    "completed": []
  }
}
```

### Структура элементов

**Предмет в сундуке / на стойке:**

```json
{
  "item_id": "dagger",
  "rarity": "rare"
}
```

**Запись квеста:**

```json
{
  "id": "dead_fields_clear",
  "progress": 0,
  "target": 1
}
```

### Статистика (`stats`)

Глобальная накопительная статистика игрока (общая для всех персонажей). Все ключи-словари — по id
(`mob_id` / `item_id` / `zone_id`); set-поля хранят только факт.

| Поле                | Тип                   | Что считает                                                        |
|---------------------|-----------------------|--------------------------------------------------------------------|
| `mobs_encountered`  | `string[]`            | встречал в бою (вкл. боссов и саммонов; ставится при появлении на поле) |
| `mobs_killed`       | `{ id: число }`       | убил (боссы и саммоны — тут же; смена фазы босса не считается)      |
| `items_crafted`     | `{ id: число }`       | улучшил редкость предмета у кузнеца (разборка не считается)         |
| `items_carried_out` | `{ id: число }`       | предмет попал в сундук из экспедиции (лут + награда-выбор за босса) |
| `items_discovered`  | `string[]`            | предмет выпал хоть раз с любого врага                              |
| `zones_entered`     | `{ zone_id: число }`  | заходов в зону (без учёта «Продолжить поиски» — это та же экспедиция) |
| `zones_returned`    | `{ zone_id: число }`  | выходов живым (победа над боссом зоны)                              |

Любая мутация статов шлёт `EventBus.emit('stats_changed')`. Часть квестов засчитывается по этим полям через
`QuestDef.condition` — в т.ч. задним числом при выдаче (`quest_granted`). См. `docs/quests.md`.

### Что сохраняется при гибели vs при победе над боссом

Досрочного выхода из экспедиции нет — она идёт до боя с боссом или до гибели. Весь лут уходит в сундук
по ходу боёв, поэтому от исхода зависит только награда-выбор за босса и фиксация прогресса зоны.

| Данные                  | Гибель             | Победа над боссом                     |
|-------------------------|--------------------|----------------------------------------|
| Золото / эссенция / лут | ✓ (уже в сундуке)  | ✓                                      |
| Сундук                  | ✓                  | ✓                                      |
| Стойки для брони        | ✓                  | ✓                                      |
| Награда-выбор за босса  | ✗ (не выдаётся)    | ✓ (draft 1 из N из `boss.loot`)        |
| Прогресс области        | ✗ (не засчитан)    | ✓ (зона в `completed_areas`)           |
| Области / квесты        | ✓                  | ✓                                      |

### Инициализация

При первом запуске (ключ отсутствует) — создаётся структура с нулями, тремя пустыми стойками для брони и
единственной открытой стартовой областью (`dead-fields`, зона магической фракции) в `unlocked_areas`. Две
другие стартовые зоны открываются покупкой пропуска у скупщика. Сразу выдаётся квест `dead_fields_clear`.
