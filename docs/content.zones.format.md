# Формат зон

Все поля в json — snake_case (например `mob_id`, а не `mobID`; `item_id`, а не `itemId`).

Файл `src/zones/{zone-id}/config.json`. Босс — это просто ссылка на моба (`mob_id`) + награда за финальный бой;
сам моб-босс и его спец-механики описаны в [формате мобов](content.mobs.format.md).

```json
{
  "id": "...",
  "name": "...",
  "description": "...",
  "faction": "...",
  "star": 1,
  "fights": { "min": 5, "max": 9 },
  "mob_pool": [
    { "mob_id": "...", "weight": 60 },
    { "mob_id": "...", "weight": 40 }
  ],
  "mob_loot": {
    "gold": { "min": 0, "max": 5 },
    "items": [
      { "item_id": "...", "chance": 0.2, "rarity": "uncommon" }
    ]
  },
  "boss": {
    "mob_id": "...",
    "loot": {
      "gold": { "min": 20, "max": 35 },
      "items": [
        { "item_id": "...", "chance": 0.5, "rarity": "uncommon" }
      ],
      "essence": { "common": { "min": 20, "max": 35 }, "uncommon": { "min": 10, "max": 10 } }
    }
  }
}
```

- `description` — необязательное краткое лор-описание; показывается в тултипе карты под названием
  (само название окрашено в цвет эссенции-награды зоны, Поле Битвы — легендарный цвет).
- `mob_pool[].mob_id` — взвешенный пул рядовых мобов зоны.
- `mob_loot` — **общий лут зоны**: выпадает с любого убитого рядового моба. Лут не привязан к конкретному мобу,
  поэтому одного и того же моба можно переиспользовать в разных зонах с разной добычей.
- `boss` — просто ссылка на моба-босса (`mob_id`) + награда за финальный бой (`loot`). Мультиатака, призыв и фазы
  босса описываются в его собственном конфиге моба, а не здесь.

`mob_loot` и `boss.loot` — таблицы лута одинакового формата: `gold: { min, max }` и `items[]`, где у каждого
элемента `item_id`, `chance` (0..1) и необязательный `rarity` (редкость, по умолчанию `common`).

- **`mob_loot`** — обычный дроп: `chance` = независимый шанс выпадения предмета. Всё выпавшее уходит в сундук
  автоматически (рюкзака и ручного подбора нет).
- **`boss.loot`** — **награда-выбор (draft из 5 карточек)**: после победы над боссом игрок берёт **одну** из 5:
  золото (`boss.loot.gold`), предмет из `mob_loot.items`, центральный предмет из `boss.loot.items`, ещё предмет
  из `mob_loot.items`, эссенция (`boss.loot.essence`). В карточках-предметах `chance` работает как **вес**
  обязательного выбора, а не как независимый шанс дропа. Недостающий источник заменяется золотом-заглушкой (42).
  Подробности — `docs/mechanics.md`, §«Награда за босса».

`boss.loot.essence` — необязательное поле: словарь по тирам эссенции (`uncommon`/`rare`/`epic`), у
каждого `{ min, max }`. При выборе карточки эссенции роллятся **все** заданные тиры, каждый в своём диапазоне.

**Канон: `boss.loot.items` = зеркало `mob_loot.items`, редкость на тир выше.** Босс не вводит новые предметы —
его список item’ов **тот же набор**, что и у рядовых мобов зоны (тот же `item_id`/`chance`), но с `rarity` на
одну ступень выше того, что фактически падает с рядовых (обычно `common` → `uncommon`). Так все три карточки
предметов драфта берутся из одного пула зоны: два боковых — рядовая редкость из `mob_loot`, центральный —
гарантированно на тир лучше. См. `src/zones/{dead-fields,trampled-meadows,armor-dump}/config.json`.

## Endless-зона (центр)

Единственная такая зона — `battlefield`. Режим включается явным полем `endless` (в отличие от boss-only, это
**не** выводится из формы данных — endless-зона не имеет `boss` вовсе):

```json
{
  "id": "battlefield",
  "name": "Поле Битвы",
  "faction": "all",
  "star": 3,
  "mob_pool": [
    { "mob_id": "...", "weight": 100 }
  ],
  "mob_loot": {
    "gold": { "min": 50, "max": 100 },
    "items": []
  },
  "endless": { "curse_per_fight": 8 }
}
```

- `boss` **не указывается** — финального боя нет, экспедиция не «выигрывается», а идёт до смерти или ручного
  отступления («В лагерь»). Награды-выбора (драфт из 5 карточек) в endless-зоне тоже нет.
- `mob_pool` — рядовые бои идут из него бесконечно (взвешенный рандом с повторами, как в обычной зоне) —
  предела длины у экспедиции нет.
- `mob_loot` — лут выпадает с каждого убитого моба, как и в обычной зоне (без отдельной награды в конце).
- `endless.curse_per_fight` — на сколько процентных пунктов растёт входящий по герою урон за каждый пройденный
  бой (аддитивно: после N боёв множитель = `1 + curse_per_fight × N / 100`). Цель зоны — рекордная: продержаться
  как можно дольше против растущего проклятия, а не «пройти» её (`completed_areas` для endless-зоны не
  используется — см. `MetaStore.battlefield_best_depth`, `docs/meta-progression.md`).

Гейт открытия центра (после прохождения всех 9 обычных зон) и отображение результата на карте — это логика
`CampScene`/`MetaStore`, не поле конфига зоны.
