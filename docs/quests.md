## Засчёт через статистику

Часть квестов завязана не на разовое событие, а на глобальную статистику (`stats` в мете — см.
`docs/meta-progression.md`). У такого квеста в `src/quests/definitions.ts` задаётся
`condition: { stat, id?, count? }`, и `QuestSystem` сверяет его автоматически:

- при изменении статов (`stats_changed`) — по ходу игры;
- при выдаче квеста (`quest_granted`) — **задним числом**: если игрок выполнил цель заранее, квест
  засчитывается сразу при появлении.

Сейчас на статистику переведены:

- зональные (`<zone>_clear`) → `{ stat: 'zones_returned', id: '<zone_id>' }`. `zones_returned`
  растёт только при полном зачёте зоны (добит босс → `finishExpedition`); ретрит не считается,
  поэтому условие эквивалентно «убить босса локации». Завязка на зону, а не на `mob_id`,
  устойчивее: некоторые боссы делят `mob_id` (напр. `beast-lair` и `predator-pasture` — оба `rat`).

Остальные (обучающие, `find_uncommon`) пока работают на событиях `EventBus` —
их статистикой по id не покрыть (редкость / дубликаты / обучение).

## Маркер на карте

У `QuestDef` есть поле `areas?: string[]` — список id зон (из `MAP_ZONE_LAYOUT` в `CampScene.ts`),
в которых можно выполнить квест. Пока не выполненный (активный) квест с непустым `areas` рисует
значок «?» в углу узла соответствующей зоны на карте — независимо от того, куплена ли проходка
(зональные квесты выдаются как `next` до покупки, значок сигналит «здесь есть задание»). У
зональных квестов `areas` дублирует `condition.id` — то же значение, но это отдельное поле:
`condition` двигает прогресс квеста, `areas` только для отображения на карте.

## Обучающие квесты

Выдаются автоматически, последовательно. Не привязаны к NPC.

| ID                     | Триггер                       | Цель                                      | Награда                          |
|------------------------|-------------------------------|-------------------------------------------|----------------------------------|
| `tutorial_equip`       | Первый предмет попал в рюкзак | Надеть любой предмет                      | 10g                              |
| `tutorial_sell`        | Есть предметы в сундуке       | Продать предмет скупщику                  | 15g                              |
| `tutorial_disassemble` | После первой продажи          | Разобрать предмет у кузнеца               | 10g + рецепт `recipe_bent_blade` |
| `tutorial_escape`      | Начата первая экспедиция      | Успешно уйти из экспедиции (побег удался) | 20g                              |

## Зональные квесты

Один квест на каждую из 9 областей (центр — Поле битвы — квеста не имеет, открывается по
топологии, когда зачищены все 9). Общий вид:

- **Название:** `Исследовать область: <название зоны>`
- **Цель:** победить босса локации (`condition: { stat: 'zones_returned', id: '<zone_id>' }`)
- **Награда:** ½ стоимости проходки в следующую зону (напр. → mage-ruins 500 ⇒ 250g;
  → crypt 5000 ⇒ 2500g). Конечные зоны маршрутов (нет следующей покупаемой) — ½ своей
  цены = 2500g. Итог: 250 / 2500 / 2500 по каждому маршруту.
- **Открытие следующей зоны:** отдельного эффекта в награде нет. Добивание босса вызывает
  `completeArea`, после чего следующая зона маршрута проходит prereq и у неё появляется кнопка
  **«Купить проходку за Ng»** (покупка входа за золото — не бесплатно).
- **`next`:** выдаёт квест следующей зоны маршрута.

Особый случай — `dead_fields_clear`: помимо 250g бесплатно открывает Растоптанные луга и
Свалку доспехов (`unlock_area`) и выдаёт их квесты (`next`). Это единственная точка входа
во 2-й и 3-й маршруты.

Стартовые квесты маршрутов:

- `dead_fields_clear` — выдан с начала игры (Мёртвые поля открыты бесплатно);
- `trampled_meadows_clear`, `armor_dump_clear` — выдаются как `next` от `dead_fields_clear`.
  Прохождение Мёртвых полей **бесплатно открывает** обе зоны (`unlock_area` в награде
  `dead_fields_clear`) и выдаёт их квесты. До этого зоны заблокированы (prereq = `dead-fields`,
  купить нельзя), покупки проходки для них больше нет.

Цепочки (по `FACTION_ROUTES`):

```
Маршрут 1: dead_fields_clear → mage_ruins_clear → crypt_clear
Маршрут 2: trampled_meadows_clear → beast_lair_clear → predator_pasture_clear
Маршрут 3: armor_dump_clear → abandoned_camp_clear → marauder_lair_clear
```

Конечные зоны маршрутов (`crypt`, `predator-pasture`, `marauder-lair`) ведут к Полю битвы —
у них нет `next`, проходка в центр не покупается (гейт по зачистке всех 9 зон).

## Коллекционные квесты

## Схема прогрессии через квесты

```
Старт
 │
 ├─ [tutorial_*] — обучение механикам (параллельно с основным прогрессом)
 │
 └─ dead_fields_clear → mage_ruins_clear → crypt_clear ┐
      │ (открывает бесплатно ↓)                          ├─ зачистка всех 9 → Поле битвы
      ├─ trampled_meadows_clear → beast_lair_clear → predator_pasture_clear ┤
      │                                                  │
      └─ armor_dump_clear → abandoned_camp_clear → marauder_lair_clear ─────┘
```
