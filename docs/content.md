# Контент игры

## Персонаж

В игре один играбельный персонаж — **Силач**. Выбора персонажа перед экспедицией нет; связанные с выбором
механики (размер рюкзака, шанс крафта в походе, бонусы к золоту/эссенции) удалены.

Базовое здоровье героя — **20** (см. [`mechanics.md`](mechanics.md)). Все прочие боевые параметры задаются
только предметами.

Экипировка героя хранится на **трёх стойках для брони** в лагере — сохранённых пресетах билдов под разные
оси фракций (см. [`factions.md`](factions.md)). Игрок выбирает активную стойку (на стенде или прямо в бою);
последний выбор запоминается и используется при входе на локацию.

## Карта и области

Карта разделена на неравномерные области. Открытие идёт от краёв к центру.

### Структура

- **Края карты**: 3 стартовые области (по одной от каждой фракции). Сразу открыта только зона магической
  фракции (Мёртвые поля); две другие стартовые зоны покупаются у скупщика за пропуск (50g)
- **Центр карты**: финальная область — единственная схватка с последним боссом (без рядовых боёв и
  без лута). Открывается автоматически после прохождения всех девяти зон по краям карты
- **Промежуточные области**: открываются постепенно по мере прохождения

### Статусы областей

- Заблокирована (нужен пропуск; для центра — прохождение всех 9 зон)
- Открыта (доступна для вылазки)
- Пройдена (босс убит, квест выполнен)

### Все области (10 зон)

Визуальная схема: `docs/concept_map.svg`

## Мобы

Каждый моб привязан к конкретным областям.

### Баланс — отдельная итерация (placeholder-статы)

Пока идёт **создание** контента, мобы не балансятся поштучно. На этом этапе каждый моб получает единые
заглушечные статы:

- `health: 10`
- `attack_damage: 5` — при мультиатаке каждая полоска атаки тоже `5`
- `attack_interval: 2.0`

Исключение — только там, где число **определяет механику**, а не баланс (например, моб-«стена» не атакует:
`attack_damage: 0` / без `attacks[]`).

Реальная балансировка — HP, урон, интервалы, кривая по тирам зон — **отдельная итерация после того, как
созданы ВСЕ мобы всех зон**. До неё числовые различия между мобами в описаниях зон не фиксируем: идентичность
моба несут его **механика и флейвор**, а не цифры. Различия вроде «быстрый / медленный», «хрупкий / танк» —
это замысел для балансировки, он живёт в тексте, а не в placeholder-числах.

### Структура конфига моба

Формат конфига моба — в отдельном файле: [`content.mobs.format.md`](content.mobs.format.md)
(базовые поля, мультиатака `attacks[]`, призыв `summons`, фазы `phases`). Босс — это просто моб, поставленный
финальным боем зоны; его спец-механики описаны в его же конфиге. Детали механик конкретных боссов — в файлах
зон (`docs/zones/`).

### Конфиг зоны

Формат конфига зоны — в отдельном файле: [`content.zones.format.md`](content.zones.format.md)
(`mob_pool`, `boss` = ссылка на моба + награда).

### Конфиг предмета

Каждый предмет = `src/items/{item_id}/config.json` + `behavior.ts` + `icon.svg`.

`config.json` хранит только идентичность, экономику и `tags` (`id`, `name`, `slots`, `type`,
`base_value`, `tags`). Боевые статы и их скейл по редкости целиком в `behavior.ts` — фабрики
`standardWeapon`/`standardArmor`/`standardShield` (единый источник правды, см.
[`combat-events.md`](combat-events.md) §5). Примеры JSON ниже **устарели** (показывают снятые поля
`attack_damage`/`damage_reduction`/`block_chance`) — фактический формат см. в `src/items/*/config.json`.

**Оружие:**

```json
{
  "id": "broken_staff",
  "name": "Обломок посоха",
  "slots": [
    "hand_right"
  ],
  "type": "weapon",
  "base_value": 10,
  "attack_damage": 1,
  "attack_interval": 3.0
}
```

**Броня:**

```json
{
  "id": "robe_scrap",
  "name": "Лоскут мантии",
  "slots": [
    "body_upper"
  ],
  "type": "armor",
  "base_value": 10,
  "damage_reduction": 1
}
```

**Щит:**

```json
{
  "id": "plank_shield",
  "name": "Щит из досок",
  "slots": [
    "hand_left"
  ],
  "type": "shield",
  "base_value": 10,
  "block_chance": 0.25
}
```

**Аксессуар** (вся механика в `behavior.ts`):

```json
{
  "id": "cracked_amulet",
  "name": "Треснувший амулет",
  "slots": [
    "body_upper"
  ],
  "type": "accessory",
  "base_value": 10
}
```

## Предметы

### Типы слотов

- `head` — голова
- `body_upper` — верх тела; сюда же идут аксессуары (амулеты, цепи, талисманы)
- `body_lower` — низ тела
- `legs` — ноги
- `hand_left` / `hand_right` — руки

Предмет занимает **ровно один слот**, но может **подходить к нескольким** — в конфиге `slots` перечислены
допустимые слоты, и игрок выбирает, в какой надеть. Пример: аксессуар со `slots: ["body_upper", "body_lower"]`
можно носить в любом из этих слотов, но только в одном за раз.

### Ценностная шкала

Базовая стоимость предмета зависит от уровня зоны где он выпадает:

| Зоны          | Common | Legendary     |
|---------------|--------|---------------|
| ★☆☆ стартовые | ~10g   | ~200–300g     |
| ★★☆ средние   | ~100g  | ~2000–3000g   |
| ★★★ финал     | ~1000g | ~20000–30000g |

Каждый тир зоны увеличивает ценность предметов в **10 раз**.

### Принципы лута стартовых зон (★☆☆)

- Лут — исключительно снаряжение с поля боя. Никаких биологических трофеев с мобов (когти, кости, шкуры).
- Предметы стартовых зон — "сломанные" или изначально слабые версии: другой `item_id`, сниженные характеристики, особый
  UI.
- **Кап редкости для сломанных предметов: `uncommon`**. Крафт выше этого порога заблокирован.
- Три стартовые зоны в совокупности знакомят игрока со всеми типами слотов экипировки.
- Боевого оружия в стартовых зонах нет — только обломки, части и слабые аналоги.

### Редкость предметов

| Редкость  | Цвет       | Название    |
|-----------|------------|-------------|
| common    | Белый      | Обычный     |
| uncommon  | Зелёный    | Необычный   |
| rare      | Синий      | Редкий      |
| epic      | Фиолетовый | Эпический   |
| legendary | Оранжевый  | Легендарный |

Параметры предмета масштабируются с редкостью через `scaleByRarity(base, rarity, scale)`.

### Предметы стартовых зон (★☆☆)

Все предметы стартовых зон — сломанные или изначально слабые версии снаряжения.
Кап редкости: `uncommon`.
Вместе три зоны знакомят игрока со всеми шестью слотами экипировки.

Описания предметов — в файлах зон:

- [Мёртвые поля](zones/dead-fields.md) — `hand_right`, `body_upper`, `body_upper` (аксессуар)
- [Растоптанные луга](zones/trampled-meadows.md) — `head`, `legs`, `hand_right`
- [Свалка доспехов](zones/armor-dump.md) — `hand_right`, `hand_left`, `body_lower`

## Крафт

Крафт — только улучшение и разборка, у кузнеца в лагере (детали и экономика — в [`mechanics.md`](mechanics.md)):

- 1 предмет + эссенция = тот же предмет следующей редкости (улучшение; потолок — эпик)

Рецептов (комбинирование двух предметов в новый) больше нет — `recipes.json`, `unlock_recipe`-квесты и
намёк «?» удалены. Новое снаряжение только находится (лут зон, награда за боссов/квесты).

## Лагерь

### Объекты лагеря

**Костёр** — у костра стоит герой (Силач), готовый к вылазке. Отсюда игрок отправляется в экспедицию,
предварительно выбрав одну из трёх стоек для брони.

**Кузнец**

- Крафт (улучшение / разборка) с 100% шансом успеха
- При открытии: левая панель = интерфейс кузнеца, правая = сундук (выбор предметов для крафта)
- Наведение на предмет в сундуке показывает сколько эссенции принесёт разборка

**Скупщик**

- Покупает предметы за золото, продаёт пропуска в новые локации
- Выдаёт квесты (по лору — торгует информацией)
- При открытии: левая панель = интерфейс скупщика, правая = сундук
- Наведение на предмет показывает цену продажи, клик = продажа

**Сундук / Стойки для брони** — единый интерфейс

- Сундук: общее хранилище снаряжения, вкладки по типам, сортировка
- Стойки для брони: три сохранённых пресета экипировки; управление тем, в чём герой пойдёт в вылазку
- Предметы из сундука можно перенести только на стойки для брони (рюкзака нет)
