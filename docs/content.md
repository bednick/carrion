# Контент игры

## Персонаж

В игре один играбельный персонаж — **Силач**. Выбора персонажа перед экспедицией нет; связанные с выбором
механики (размер рюкзака, шанс крафта в походе, бонусы к золоту/эссенции) удалены.

Базовое здоровье героя — **100** (см. [`mechanics.md`](mechanics.md)). Все прочие боевые параметры задаются
только предметами.

Экипировка героя хранится на **трёх стойках для брони** в лагере — сохранённых пресетах билдов под разные
оси фракций (см. [`factions.md`](factions.md)). Игрок выбирает активную стойку (на стенде или прямо в бою);
последний выбор запоминается и используется при входе на локацию.

## Карта и области

Карта разделена на неравномерные области. Открытие идёт от краёв к центру.

### Структура

- **Края карты**: 3 стартовые области (по одной от каждой фракции). Сразу открыта только зона магической
  фракции (Мёртвые поля); две другие стартовые зоны покупаются у скупщика за пропуск (50g)
- **Центр карты**: финальная область — единственная схватка с последним боссом (без рядовых боёв и
  без лута). Открывается автоматически после прохождения всех девяти зон по краям карты
- **Промежуточные области**: открываются постепенно по мере прохождения

### Статусы областей

- Заблокирована (нужен пропуск; для центра — прохождение всех 9 зон)
- Открыта (доступна для вылазки)
- Пройдена (босс убит, квест выполнен)

Визуальная схема: `docs/concept_map.svg`

## Мобы

Каждый моб привязан к конкретным областям.

### Баланс — здоровье и опорный УВС

Контент всех 10 зон создан — числовой баланс мобов больше не плейсхолдер, а считается по правилу ниже.
Интервалы атак (`attacks[].interval`) остаются флейвором конкретного моба (быстрый/медленный, как задумано в
описании зоны) и правилом не переопределяются — правило задаёт только `health` и `attacks[].damage`.

**HP по тиру моба** — единая шкала, одинаковая для всех зон независимо от звёздности:

| Тир | HP |
|---|---|
| Вводный (костяк 3 стартовых зон) | 25 |
| Обычный | 50 |
| Сильный | 75 |
| Босс (за фазу) | 100 |

Тир мобу внутри `mob_pool` зоны присваивается по весу спавна (топ вес → слабее, низкий вес → сильнее); тир
`вводный` (25) существует только в трёх стартовых зонах — «первое знакомство» с игрой, у средних зон низший
тир пула уже `обычный` (50).

**Число боёв в зоне** (`fights.min/max` в конфиге зоны) фиксировано по группам, а не одинаково для всех:

| Группа | Зоны | Боёв (рядовых) |
|---|---|---|
| Стартовые (спец-правила, не по фракции) | Мёртвые поля, Растоптанные луга, Свалка доспехов | 5–7 |
| Люди / звери | Логово мародёров, Брошенный лагерь, Логово зверей, Пастбище хищников | 9–11 |
| Нежить (истощение) | Руины магов, Склеп | 13–15 |
| Финал | Поле Битвы | 0 (только босс) |

Мёртвые поля — зона нежити, но как стартовая зона живёт по «стартовым» правилам (5–7, не 13–15), так же и
стартовые зоны двух других фракций.

**Целевой остаток HP героя.** Два билд-якоря, оба сейчас проверяемы (★☆☆ и ★★☆):

- **Стартовые зоны — якорь «безоружный игрок».** В самом начале игры на стойке героя лежит только `battle_staff`
  (`common`), без брони и щита (`src/core/MetaStore.ts`, `defaultStands()`) — это гарантированный первый билд
  каждого нового игрока, а не гипотетический нижний край. Билд — `tools/builds/basic-start.json`. Стартовые зоны
  обязаны быть проходимы **им одним**: доходить до босса нужно почти всегда, но с ощутимо подъеденным HP (не
  «легко»). Ориентир по факту (после калибровки симулятором) — winrate ~90%+ с этим билдом, HP перед боссом
  ~60–75%, после победы — уже правда мало (~10–35%, «на волоске»).
- **Средние зоны — якорь «фракционный билд, две стадии редкости».** Формульные 50%/75% (см. формулу ниже) были
  изначальным планом, затем заменены на build-anchor подход — но **не одним общим билдом на все 6 зон** (первая
  попытка так и делала и была снята как методологически неверная, см. [`roadmap.balance.md`](roadmap.balance.md)
  §«Средние зоны — почему первый заход был неверным»), а **тремя фракционными билдами** (Мародёры/Звери/Нежить,
  `docs/factions.md`), каждый в **двух стадиях редкости**: `uncommon` на входе в первую среднюю зону фракции,
  полный 4-слотовый билд `rare` на входе во вторую. Билд-файлы — `tools/builds/{marauders,beasts,undead}-
  uncommon.json`/`-rare-full.json`. Ориентир по факту — winrate 85–95% на каждую зону отдельно (не агрегат), HP
  перед боссом 32–54%, после победы 12–39% (один особый случай — `crypt`, см. `roadmap.balance.md`).

Формульный подход остаётся справочным (ниже) — полезен как первая прикидка перед прогоном симулятора, но
финальные числа во всех зонах, где баланс сделан, взяты из Monte-Carlo с конкретным билдом, не из формулы напрямую.

**Формула УВС.** Опорный DPS героя зоны — справочная величина, не хранится в конфиге:

```
hero_DPS_ref(zone) = weighted_avg_HP(mob_pool) / TTK_target(zone)
```

УВС рядового моба — одинаков для всех тиров внутри зоны (тир кодирует только живучесть, не урон/сек); при своре
(несколько тел разом, как `rat`) бюджет делится между одновременными телами:

```
mob_DPS(zone) = 50 / (N_боёв × TTK_target(zone))
```

УВС босса — весь бой, все фазы вместе, как единый бюджет относительно суммарного HP босса:

```
boss_total_HP = 100 × число_фаз
boss_DPS(zone) = 25 × hero_DPS_ref(zone) / boss_total_HP
```

`attack_damage` конкретной полоски = `round(доля_DPS_на_полоску × interval_полоски)`.

⚠ **Важная поправка, найденная эмпирически** (через `npm run balance`, см. ниже): формула выше даёт **сырой**
урон без учёта брони героя. Броня (`damage_reduction` предметов, с 2026-07 — процентная и мультипликативная, см.
[`mechanics.md`](mechanics.md) §«Броня vs щит») ощутимо съедает средний DPS мобов (см.
[`content.mobs.format.md`](content.mobs.format.md) §«Защита»): при типичной для тира броне игрока
итоговый урон мобов приходится подбирать **выше** чистого расчёта, эмпирически, через симулятор — просто
следовать формуле без проверки боем даёт нереалистично мягкий (или, при грубой перекомпенсации, нереалистично
жёсткий) результат. Финальные числа трёх стартовых зон — в `src/mobs/*/config.json` и `docs/zones/{зона}.md`,
подобраны и провалидированы через Monte-Carlo симулятор (см. ниже), не только формулой.

**Проверка.** Есть готовый Monte-Carlo инструмент, использующий настоящий боевой движок:
`npm run balance -- <zoneId> --trials=5000 --build=<путь>`. Три стартовых зоны провалидированы на **обоих**
билдах:

- `tools/builds/basic-start.json` (только `battle_staff` `common`, без брони/щита — реальный старт новой игры)
  — основной якорь калибровки для этой тройки зон.
- `tools/builds/dead-fields-1/2/3-common.json` (варианты `common`-оружия из «Мёртвых полей») — вспомогательные
  тестовые сборки.

> Прежние версии этого раздела ссылались на `tools/builds/fresh-start.json`/`basic-uncommon.json` — этих файлов
> по факту не существует в `tools/builds/` (стале-заметка, не сверенная с реальным содержимым каталога).

Своры (несколько одновременно атакующих тел, как `rat`) заметно повышают дисперсию исхода относительно
одиночного моба той же средней DPS — второе тело продолжает атаковать всё время, пока герой добивает первое,
поэтому суммарная экспозиция вдвое больше, чем у одиночного моба такой же живучести. В `trampled-meadows`
`rat`/`giant_rat` из-за этого держат HP **ниже** формульных тиров (8 и 25 вместо 25 и 50) — подобрано
эмпирически через симулятор, не по чистой математике, см. предупреждения в `docs/zones/trampled-meadows.md`.

**Сделано (2026-07):** все 6 средних зон (`abandoned-camp`, `marauder-lair`, `beast-lair`, `crypt`, `mage-ruins`,
`predator-pasture`) пересчитаны и провалидированы Monte-Carlo — `health`/`attacks[].damage`/`defense`/`summons`
в `src/mobs/*/config.json`, синхронная проза в `docs/zones/{зона}.md`. Якорь калибровки — не формульные 50%/75%
и не единый билд на все зоны (первая попытка так и делала, была неверной и полностью переделана), а **3
фракционных билда × 2 стадии редкости** (Мародёры/Звери/Нежить, `uncommon` → `rare`-полный), см.
[`roadmap.balance.md`](roadmap.balance.md) §«Средние зоны — сделано (2026-07, второй заход)» — там же метод
подбора билдов и разбор находки про циклическую структуру лут-таблиц фракций. Результат по 5 из 6 зон: винрейт
85–95% на каждую отдельно, остаток HP перед боссом 32–54%, после победы 12–39% — «на грани, но проходимо».
`crypt` — особый случай (билд с dual-кинжалом и `leech_bead` самолечит рядовые бои, бой с боссом детерминирован
без вероятностного разброса) — откалиброван на остаток HP после победы (~29%), не на литеральный винрейт, см.
roadmap. Вне объёма этого прохода: ребаланс уже реализованных предметов (`hand_right`/`hand_left`/`body`/
`amulet`), слот `legs` (не реализован), домены `head`/`ring` (не спроектированы). `battlefield.md` (★★★,
финальный босс) — мобы там вообще не заведены в `src/mobs`, это контентная, не балансовая задача.

### Структура конфига моба

Формат конфига моба — в отдельном файле: [`content.mobs.format.md`](content.mobs.format.md)
(базовые поля, мультиатака `attacks[]`, призыв `summons`, фазы `phases`). Босс — это просто моб, поставленный
финальным боем зоны; его спец-механики описаны в его же конфиге. Детали механик конкретных боссов — в файлах
зон (`docs/zones/`).

### Конфиг зоны

Формат конфига зоны — в отдельном файле: [`content.zones.format.md`](content.zones.format.md)
(`mob_pool`, `boss` = ссылка на моба + награда).

### Конфиг предмета

Каждый предмет = `src/items/{item_id}/behavior.ts` + `icon.svg`. Отдельного `config.json` нет —
идентичность и экономика (`name`, `slots`, `type`, `baseValue`, `tags`) объявлены прямо на объекте
`ItemBehavior` в `behavior.ts`, вместе с боевыми статами и их скейлом по редкости (единый источник
правды, см. [`combat-events.md`](combat-events.md) §5). Для простых предметов идентичность
дописывается поверх фабрики (`standardWeapon`/`standardArmor`/`standardShield`) через спред.

**Оружие:**

```ts
import type { ItemBehavior } from '../behavior';
import { standardWeapon } from '../factories';

const behavior: ItemBehavior = {
  name: 'Боевой посох',
  slots: ['hand_right'],
  type: 'weapon',
  baseValue: 10,
  tags: ['weapon', 'heavy', 'slow'],
  ...standardWeapon({ damage: 5, interval: 1.0, scale: { damage: 1.5, interval: 1.0 } }),
};

export default behavior;
```

**Броня:**

```ts
import type { ItemBehavior } from '../behavior';
import { standardArmor } from '../factories';

const behavior: ItemBehavior = {
  name: 'Блестящие латы',
  slots: ['body'],
  type: 'armor',
  baseValue: 10,
  tags: ['armor'],
  ...standardArmor({ pct: 0.1 }),
};

export default behavior;
```

**Щит и аксессуар** — та же форма: `ItemBehavior`-объект с идентичностью (`name`/`slots`/`type`/
`baseValue`/`tags`) и боевыми хуками (`on`, `stats`, …), которые для нестандартной механики пишутся
руками поверх фабрики или вместо неё — см. `src/items/heavy_shield/behavior.ts`,
`src/items/vulture_amulet/behavior.ts`.

## Предметы

### Типы слотов

- `head` — голова
- `body` — тело (торс)
- `legs` — ноги
- `hand_left` / `hand_right` — руки
- `ring` — кольцо
- `amulet` — амулет; сюда же исторически шли аксессуары-подвески (цепи, талисманы)

Предмет занимает **ровно один слот**, но может **подходить к нескольким** — в конфиге `slots` перечислены
допустимые слоты, и игрок выбирает, в какой надеть. Пример: аксессуар со `slots: ["ring", "amulet"]`
можно носить в любом из этих слотов, но только в одном за раз.

### Ценностная шкала

Базовая стоимость предмета зависит от уровня зоны где он выпадает:

| Зоны          | Common | Legendary     |
|---------------|--------|---------------|
| ★☆☆ стартовые | ~10g   | ~200–300g     |
| ★★☆ средние   | ~100g  | ~2000–3000g   |
| ★★★ финал     | ~1000g | ~20000–30000g |

Каждый тир зоны увеличивает ценность предметов в **10 раз**.

### Принципы лута стартовых зон (★☆☆)

- Лут — исключительно снаряжение с поля боя. Никаких биологических трофеев с мобов (когти, кости, шкуры).
- Предметы стартовых зон — "сломанные" или изначально слабые версии: другой `item_id`, сниженные характеристики, особый
  UI.
- **Кап редкости для сломанных предметов: `uncommon`**. Крафт выше этого порога заблокирован.
- Три стартовые зоны в совокупности знакомят игрока с частью типов слотов экипировки (не всеми — `ring`
  ещё не имеет своего дропа в стартовых зонах).
- Боевого оружия в стартовых зонах нет — только обломки, части и слабые аналоги.

### Редкость предметов

| Редкость  | Цвет       | Название    |
|-----------|------------|-------------|
| common    | Белый      | Обычный     |
| uncommon  | Зелёный    | Необычный   |
| rare      | Синий      | Редкий      |
| epic      | Фиолетовый | Эпический   |
| legendary | Оранжевый  | Легендарный |

Параметры предмета масштабируются с редкостью через `scaleByRarity(base, rarity, scale)`.

### Предметы стартовых зон (★☆☆)

Все предметы стартовых зон — сломанные или изначально слабые версии снаряжения.
Кап редкости: `uncommon`.

Раскладка лута по зонам (R10/R11, `content.items.md`) — только в `src/zones/*/config.json`, не здесь. Описания
зон:

- [Мёртвые поля](zones/dead-fields.md)
- [Растоптанные луга](zones/trampled-meadows.md)
- [Свалка доспехов](zones/armor-dump.md)

## Крафт

Крафт — только улучшение и разборка, у кузнеца в лагере (детали и экономика — в [`mechanics.md`](mechanics.md)):

- 1 предмет + эссенция = тот же предмет следующей редкости (улучшение; потолок — эпик)

Рецептов (комбинирование двух предметов в новый) больше нет — `recipes.json`, `unlock_recipe`-квесты и
намёк «?» удалены. Новое снаряжение только находится (лут зон, награда за боссов/квесты).

## Лагерь

### Объекты лагеря

**Костёр** — у костра стоит герой (Силач), готовый к вылазке. Отсюда игрок отправляется в экспедицию,
предварительно выбрав одну из трёх стоек для брони.

**Кузнец**

- Крафт (улучшение / разборка) с 100% шансом успеха
- При открытии: левая панель = интерфейс кузнеца, правая = сундук (выбор предметов для крафта)
- Наведение на предмет в сундуке показывает сколько эссенции принесёт разборка

**Скупщик**

- Покупает предметы за золото, продаёт пропуска в новые локации
- Выдаёт квесты (по лору — торгует информацией)
- При открытии: левая панель = интерфейс скупщика, правая = сундук
- Наведение на предмет показывает цену продажи, клик = продажа

**Сундук / Стойки для брони** — единый интерфейс

- Сундук: общее хранилище снаряжения, вкладки по типам, сортировка
- Стойки для брони: три сохранённых пресета экипировки; управление тем, в чём герой пойдёт в вылазку
- Предметы из сундука можно перенести только на стойки для брони (рюкзака нет)
