# Система предметов: слоты и теги

Функциональное описание того, **что делает** экипировка: за какой домен отвечает каждый слот, какие архетипы доступны
внутри слота и как теги связывают эффекты предметов в синергии. Это справочник по функциональности (не по реализации).
Канон формата конфигов — [`content.md`](content.md) и `src/items/`; событийная модель эффектов — [
`combat-events.md`](combat-events.md).

Три опоры системы:

- **Слот = роль.** Каждый слот владеет своим доменом игровых глаголов (урон, защита, мобильность, добыча, хранение).
- **Архетип = ось выбора внутри слота.** В рамках слота предмет выбирает один из архетипов — края одной оси. Это держит
  выбор осмысленным, без «стат-супа».
- **Теги = язык синергий.** Предметы объявляют теги; эффекты и синергии считаются по **суммарному набору тегов всей
  надетой экипировки**, а не по хардкоду конкретных пар предметов.

> **📌 Охват текущей проработки.** Сейчас прорабатываем **только** слоты: `legs` (обувь), `hand_right`,
> `hand_left`, `body`, `amulet`. Слоты `head`, `ring` **не заполняем** — созданные под них предметы
> **удаляем**, домены и механики этих слотов будут **полностью спроектированы позже**. Описание `head`
> ниже оставлено как справка о прежнем замысле и **не актуально**. `body_lower` как слот **упразднён** —
> прежний домен «защищённый схрон» (рюкзак/шанс побега) удалён вместе с самим слотом, не просто заморожен.
>
> Замечание по `head`: прежний домен добычи (`magic_find`, качество/количество дропа) целиком переехал в
> `legs` — см. [`content.items.legs.md`](content.items.legs.md). После переноса слот `head` пуст и ждёт
> нового домена.
>
> `ring` — новый слот (структурно заведён, домен ещё не спроектирован), пуст. `amulet` больше не
> placeholder — формализован как домен восстановления HP, см. [`content.items.amulet.md`](content.items.amulet.md);
> `vulture_amulet` и `leech_bead` оба реально надеваются в `amulet` (старая пометка «`leech_bead` → `ring`»
> была неверна и снята этой правкой). `hand_right`, `hand_left`, `body`, `amulet` — актуальны.

## Базовые правила (R1–R11)

Рамка любой доработки предметов.

- **R1 — Слот = одна ось.** Предмет владеет одним доменом-глаголом слота. Домены не смешиваются. 7 слотов — 7 осей.
- **R2 — Архетип = полюс.** Предмет выбирает край оси слота, несёт теги-идентичность + свою кривую скейла редкости.
- **R3 — Нет noop.** У каждого предмета живой эффект. Мёртвый слот чинится **сначала** переопределением домена, потом
  предметы. Заглушка «в разработке» не шипается.
- **R4 — Два движка.** Эффект живёт ровно в одном: бой (хук события, `on`) или мета (модификатор лут/забега, `meta`).
  `stats()` в тултипе — единый источник правды по числам (бой и скупщик читают те же).
- **R5 — Редкость — единственный крафт-рычаг.** Базовый стат + кривая скейла на архетип. Без стат-супа: предмет
  скейлит свой ОДИН профильный стат (+ опциональный слабый rider). Крафт не добавляет статы, только скейлит.
- **R6 — Покрытие фракций.** Пул обязан давать каждый герой-инструмент из матрицы: сплеш/cleave, тяжёлый пробой,
  контра/мультихит, реновация, безусловная митигация. Каждый — минимум один добываемый предмет. Ворота «пул готов».
- **R7 — Осознанный минус на топ-выживании.** Топ-предметы `body` несут намеренный минус (+броня −скорость атаки).
  Рождает синергии, блокирует доминирование одного предмета.
- **R8 — Теги декларативны.** Тег — маркер («есть эффект on_kill»), реакция живёт в хуке. Новый тег-эффект ⇒ обязан
  иметь тип события в реестре (`combat-events.md` §2).
- **R9 — Всё в behavior.ts.** Идентичность (`name`, `slots`, `type`, `tags`) и вся боевая
  математика — в одном `ItemBehavior`-объекте в `behavior.ts`, отдельного `config.json` у предметов нет (в отличие
  от мобов/зон, где `config.json` остаётся и snake_case там сохраняется).
- **R10 — Лут крест-накрест (камень-ножницы-бумага).** Чемпион-инструмент против фракции X **не падает** в зонах
  фракции X — он добывается в зоне другой фракции, по фиксированному направленному циклу между тремя фракциями
  (Нежить/Звери/Мародёры). Это буквально то же «Круг замкнут» из [`factions.md`](factions.md) («Ключевые замки»),
  применённое к размещению лута: инструмент, который в зоне не работает или вредит герою (см. матрицу
  противодействия), не мешает генерировать его именно там, а нужный для этой зоны инструмент всё равно
  приходится нести с собой, добытый заранее в другом месте. Направление цикла и полная раскладка — не в этом
  доке, см. R11.
- **R11 — Зона-конфиг — единственный источник правды по размещению лута.** Какой `item_id` падает в какой зоне
  (и с какой `chance`/`rarity`) — решается и хранится **только** в `src/zones/*/config.json`
  (`mob_loot`/`boss.loot`). Файлы `content.items.*.md` описывают предмет, его архетип и фракцию-цель по матрице
  противодействия (R10), но **не** дублируют конкретные id зон, не пишут «Дом — `<zone>`» и не хранят историю
  переездов между зонами — это дублирующийся источник правды, который расходится с кодом при любой правке лута.
  При изменении раскладки по зонам меняется только конфиг зоны, документация предмета не трогается.

### Политика баланса категории

Внутри одной категории **нет предмета, ультимативно сильнее другого** — только сайдгрейды (разные цели/фракции).
Речь про **разные** предметы; более редкая версия ОДНОГО предмета законно сильнее (это ось крафта, R5). «Сильный билд
не замечает цену» — вопрос баланса общей сложности карт, а не функционала предмета.

## Слоты и роли

| Слот         | Домен                 | Архетипы (выбор внутри слота)                                                                                                    |
|--------------|-----------------------|----------------------------------------------------------------------------------------------------------------------------------|
| `hand_right` | основной урон         | **тяжёлое** (урон×качество, медленно) / **сбалансированное** (средние урон и темп) / **лёгкое** (скорость×качество, слабый удар) |
| `hand_left`  | защита / доп. урон    | **щит** (шанс блока, контрудар) / **второе оружие** (свой таймер атаки)                                                          |
| `head`       | ⏸ вне проработки      | _пусто — прежний домен добычи переехал в `legs`, ждёт нового домена_                                                             |
| `body`       | оборона (снижение урона) | открытый веер: **чистая броня** / **шипы** (rider) / **потолок урона за удар** / **реактивная** (HP-порог) / **тяжёлая** (трейд-офф скорости атаки) |
| `legs`       | форма забега          | **скороход** (короче забег, выше шанс побега) / **мародёр** (длиннее забег и лут, ниже шанс побега)                              |
| `ring`       | ⏸ вне проработки      | _новый слот, домен не спроектирован, пуст_                                                                                        |
| `amulet`     | восстановление HP     | легаси-наступательные (**лечение за удар/килл**, закрыты для нового контента) / **аварийный хил** (HP-порог, раз за бой) / **барьер** (пул поверх HP на бой) |

Принцип: каждый слот — **одна ось выбора**, архетипы — края этой оси. Список архетипов внутри слота открыт и
пополняется новым контентом. Архетип — это **идентичность** (набор тегов для синергий и тултипов) плюс собственная
кривая скейла стата от редкости; разные архетипы одного слота могут вести себя по-разному.

### `hand_right` — основной урон

> **Источник правды — [`content.items.hand_right.md`](content.items.hand_right.md).** Там домен, ось,
> формы урона (архетипы), характеристики и список предметов. Ниже — краткая выжимка.

Ось — **профиль подачи урона**: концентрация (одна цель) ↔ распределение (много целей/ударов). Темп/крит/
контроль/накопление — дескрипторы поверх (не отдельные оси, R1). Архетипы — открытый веер:

- **Тяжёлое**: высокий урон, медленный темп; урон сильно растёт с редкостью. Несёт `slow`. Анти-Мародёр.
- **Сбалансированное**: средние урон и темп, без перекоса осей.
- **Лёгкое**: низкий урон, быстрый темп; темп растёт с редкостью. Несёт `fast`. Кормит вампиризм.
- **Мультихит / крит / cleave / раскачка / стан** — формы урона под фракции; часть за фундаментами из
  [`roadmap.md`](roadmap.md) (доска 2×2, стаки, статусы). Детали — в профильном файле.

Оружие — чемпион **Зверей** (cleave/мультихит) и **Мародёров** (тяжёлое/крит), нежить намеренно не контрит
(канон: анти-нежить = реновация из другого слота, оружие лишь кормит вампиризм).

### `hand_left` — защита / доп. урон

> **Источник правды — [`content.items.hand_left.md`](content.items.hand_left.md).** Там домен, ось, три узла
> «реакции на удар», характеристики и список предметов. Ниже — краткая выжимка.

Ось — **куда идёт рука**: реакция на удар ↔ усиление урона — бинарный выбор полюса, не континуум. Слот
опционален (можно оставить пустым без штрафа).

- **Реакция на удар** — три отдельных узла, каждый с ровно одним профильным стат (не совмещаются в одном
  предмете): **чистая защита** (`block_chance` — единственный реально снижающий урон предмет слота, безусловная
  митигация против всех трёх фракций) / **контрудар** (`counter_chance` — независимый бросок на каждый входящий
  удар, урон не снижается; кормится частотой атак — Звери) / **шипы** (`thorns_ratio` — безусловный, без броска,
  доля любого дошедшего урона назад; надёжен против редких тяжёлых ударов Мародёров).
- **Усиление урона** — два разных механизма: **второй поток** (`dagger` — свой независимый таймер атаки,
  компактные light-архетипы `hand_right`) / **перчатки** (новый `type: "gloves"` — без своего таймера, три узла:
  крит-магнитуда `heavy_gloves`, устойчивая скорость `light_gloves`, быстрый первый удар боя
  `comfortable_gloves`; все усиливают уже надетое оружие `hand_right`, не добавляя параллельный источник урона).
  Ни один узел защиту слота не даёт.

### `head` — качество добычи

> **⏸ Вне текущей проработки.** Домен добычи переехал в `legs` целиком; слот пуст. Созданные под `head`
> предметы удаляются, новый домен слота проектируется позже. Ниже — прежний замысел как справка.

Управляет качеством и количеством находок. Архетип «качество» повышает шанс улучшить редкость найденных предметов
(magic find); «количество» — увеличивает число дропов / золота. Создаёт дилемму «слот под силу vs слот под добычу».

### `body` — оборона

> **Источник правды — [`content.items.body.md`](content.items.body.md).** Там домен, ось, полный список
> предметов и характеристики. Ниже — краткая выжимка.

Домен сужен этой правкой: восстановление (лечение за удар/убийство) переехало в `amulet` целиком, `body`
теперь строго про **снижение получаемого урона** — открытый веер форм от пассивной константы (чистая броня) до
условных/реактивных (шипы-rider, потолок урона за удар, реактивная защита по HP-порогу). Топовый предмет слота
несёт осознанный **минус** (+защита, −скорость атаки через кросс-slot `weaponTimerMod`, R7) — страхует от
доминирования одного предмета.

### `legs` — форма забега

> **⚠️ Ждёт переработки.** Роль опирается на шанс побега (удалён). Длина забега (число боёв) остаётся
> валидной осью, но «шанс побега» больше не существует. Ниже — прежний замысел до нового дизайна `legs`.

Обувь **модифицирует** существующие формулы мета-петли, не заменяя их:

- **Шанс побега**: аддитивный модификатор поверх штрафа за заполненность рюкзака (`+` у скорохода, `−` у мародёра).
- **Длина вылазки**: число боёв зоны ± модификатор обуви. «Мародёр» удлиняет забег (риск/награда), «скороход»
  укорачивает.

Игрок выбирает обувью **профиль риска забега**, а не только силу в бою.

### `amulet` — восстановление

> **Источник правды — [`content.items.amulet.md`](content.items.amulet.md).** Там домен, ось, полный список
> предметов, механика барьера/аварийного хила. Ниже — краткая выжимка.

Домен — восстановление HP-ресурса героя, симметрично `body` (который отвечает только за снижение урона, R1).
Два легаси-предмета (`leech_bead`, `vulture_amulet`) реагируют на урон, который **наносит** герой — этот
триггер закрыт для нового контента. Новые предметы — только защитные и только косвенно (без стата снижения
урона): **аварийный хил** (`thread_charm`, разово за бой при HP ниже порога) и **барьер** (`barrier_amulet`,
временный HP-пул поверх обычного, выдаётся заново на каждый бой).

### `ring` — новый слот

> **⏸ Вне текущей проработки.** Слот структурно заведён (тип, paper doll, диспетчер), но домен и архетипы ещё
> не спроектированы, пуст.

## Теги

Теги — **декларативные дескрипторы** (что предмет есть / делает). Их читают чужие эффекты для условных синергий, а
также тултипы, квесты и фильтры. Теги **не** механизм реакции на события: сама реакция выражается подпиской эффекта на
тип события (`kill`, `damage`…, см. [`combat-events.md`](combat-events.md)). Поэтому тег вроде `on_kill` — это
пометка-маркер «у предмета есть эффект на убийство», а эффект живёт в поведении предмета.

| Тег          | Категория        | Значение                                                                                  |
|--------------|------------------|-------------------------------------------------------------------------------------------|
| `weapon`     | категория        | оружие — даёт поток атаки (интервал + автор)                                              |
| `shield`     | категория        | щит — шанс полного блока                                                                  |
| `armor`      | категория        | броня — пассивное снижение урона                                                          |
| `accessory`  | категория        | аксессуар — эффект целиком в скрипте                                                      |
| `heavy`      | архетип оружия   | тяжёлое: высокий урон, медленный темп; урон сильно скейлит редкость. Подразумевает `slow` |
| `balanced`   | архетип оружия   | сбалансированное: средние урон и темп, без перекоса осей                                  |
| `light`      | архетип оружия   | лёгкое: низкий урон, быстрый темп; темп скейлит редкость. Подразумевает `fast`            |
| `fast`       | дескриптор темпа | низкий интервал атаки — крючок для синергий «за удар» (`lifesteal` × `fast`)              |
| `slow`       | дескриптор темпа | высокий интервал атаки — обычно у `heavy`; крючок для «за сильный удар»                   |
| `lifesteal`  | эффект-кейворд   | лечит от нанесённого урона / убийства                                                     |
| `on_kill`    | эффект-кейворд   | имеет эффект на событие `kill`                                                            |
| `on_hit`     | эффект-кейворд   | имеет эффект на нанесённый `damage`                                                       |
| `block`      | эффект-кейворд   | даёт шанс блока (на щите)                                                                 |
| `bleed`      | эффект-кейворд   | накладывает кровотечение                                                                  |
| `thorns`     | эффект-кейворд   | контрурон в ответ на получаемый урон                                                      |
| `counter`    | эффект-кейворд   | контрудар по атаковавшему за успешный блок                                                |
| `damage_cap` | эффект-кейворд   | ограничивает урон одного удара абсолютным потолком (`body`)                               |
| `last_stand` | эффект-кейворд   | эффект включается при HP героя ниже порога — доп. защита (`body`) или аварийный хил (`amulet`) |
| `barrier`    | эффект-кейворд   | даёт временный HP-пул поверх обычного на весь бой, выдаётся заново на каждый бой (`amulet`) |
| `magic_find` | мета-эффект      | повышает шанс улучшить редкость найденных предметов (голова)                              |
| `secure`     | мета-эффект      | ⚠️ мёртвый тег: защищённые ячейки схрона были в упразднённом слоте `body_lower`             |
| `escape`     | мета-эффект      | ⚠️ ждёт переработки: модификатор шанса побега (обувь) — побег удалён                      |
| `salvage`    | мета-эффект      | бонус эссенции при разборке                                                               |

Связь архетипа и темпа: `heavy ⇒ slow`, `light ⇒ fast`, `balanced` — ни то, ни другое. Архетип = идентичность
(тултипы / квесты), `fast` / `slow` = синергийные крючки темпа.

## Синергии

Эффекты складываются в билды через два уровня связи:

- **Эмерджентный** — без чтения тегов: лечение `on_hit` автоматически сильнее с `fast`-оружием, потому что ударов
  больше (синергия рождается из частоты событий, без спец-кода на конкретную пару).
- **Условный / именованный** — эффект явно проверяет чужие теги в надетой экипировке: «если надето `heavy`-оружие →
  +X». Поверх этого возможны **именованные пары** с явной подсказкой в тултипе, чтобы топовые синергии были заметны.
